# 路线图

## 当前定位

drexor-python 当前是基础设施核心后端，优先保证认证、DB、Redis、队列、调度、WebSocket 和工程质量。

## 阶段规划

| 阶段 | 目标 | 状态 |
| --- | --- | --- |
| 01 | 基础设施与架构整理：`packages/common`、路由入口、测试目录、项目更名 | ✅ |
| 02 | 文档体系：`docs/product`、`docs/architecture`、`docs/api`、`docs/dev-phase` | ✅ |
| 03 | 认证安全加固：钱包 nonce Redis 化、统一异常响应、API Key 缓存一致性 | ⬜ |
| 04 | 队列/调度落地业务样例：真实 consumer/job、启动生命周期接入 | ⬜ |
| 05 | 多 service 演进：`contracts/`、内部 client、示例独立 service | ⬜ |

## 当前已完成

- FastAPI 主应用
- PostgreSQL / Redis 初始化
- 密码登录 / 钱包登录 / TOTP / API Key
- Redis Streams 消费者框架
- APScheduler 调度框架
- WebSocket 基础连接与广播
- `packages/common`
- `tests/` 测试目录
- `drexor-python` 项目名

## 近期重点

1. 统一错误响应格式。
2. 钱包 nonce 存入 Redis，并增加过期和一次性消费。
3. 梳理 API Key 模块，确认当前 `AuthService` 与旧 `ApiKeyService` 的边界。
4. 将 scheduler / queue 的启动关闭接入 FastAPI lifespan，或明确默认禁用策略。
5. 建立 `.env.example` 与配置文档同步机制。
