# 多服务演进

## 目标

为未来可能出现的独立 service 建立边界规则，但当前不提前拆微服务。

## 进度

| 任务 | 状态 |
| --- | --- |
| `packages/common` 共享工具 | ✅ |
| `docs/architecture/README.md` 多服务演进说明 | ✅ |
| `contracts/` 目录 | ⬜ |
| `app/clients/` 内部服务 client 示例 | ⬜ |
| 示例独立 service | ⬜ |

## 决策

- 同步服务间调用优先 HTTP REST。
- 异步事件优先 Redis Streams。
- 当前不需要注册中心。
- 本地、Docker Compose、Kubernetes 分别通过 `.env`、service name、Service DNS 发现服务。
- `packages/common` 只放纯工具。
- `contracts/` 只放跨服务契约，不放业务实现。

## 触发拆 service 的条件

- 独立部署 / 扩缩容
- 独立权限或安全边界
- 资金、链上操作、密钥等高风险隔离
- 长任务 worker 和 API 生命周期明显不同
