# plan · 计划 / 设计草图

> 本目录放还未实现的设计、备选方案和 TODO。
> 已实现的产品能力沉淀到 `../product/`，开发进度记录到 `../dev-phase/`。

## 与其它文档的分工

| 目录 | 回答的问题 | 时态 |
| --- | --- | --- |
| `../product/` | 系统当前长什么样，用户和运营能做什么 | 当前已实现 |
| `../plan/` | 将来要做什么、怎么设计、还有哪些待确认 | 未来未实现 |
| `../dev-phase/` | 做了什么、做到哪、如何验证 | 进行中 / 已完成 |
| `../api/` | HTTP/WebSocket 接口怎么调用 | 当前已实现 |

## 生命周期

```text
plan/  ->  dev-phase/  ->  product/ + api/
设计草图    进度跟踪       已实现能力和接口契约
```

- 未开工的能力先写在 `plan/`。
- 开工后在 `dev-phase/` 跟踪任务和验证。
- 完成后把用户可见能力沉淀到 `product/`；涉及接口时同步 `api/`。

## 状态标记

| 符号 | 含义 |
| --- | --- |
| ✅ | 依赖项已实现 |
| 🔄 | 部分实现 / 进行中 |
| ⬜ | 未实现 |
