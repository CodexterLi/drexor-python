# 认证与用户

## 认证方式

当前支持三类认证：

| 方式 | 面向对象 | 说明 |
| --- | --- | --- |
| JWT Cookie / Bearer | 浏览器、普通 API 调用 | 登录后设置 `access_token` / `refresh_token` HttpOnly Cookie |
| 钱包签名登录 | EVM 钱包用户 | nonce + message + signature，验证成功后自动注册 |
| API Key + Secret | SDK、脚本、服务调用 | Header `X-Api-Key` + `X-Api-Secret` |

## 密码登录

流程：

```text
POST /api/auth/login
    -> AuthService.authenticate_user
    -> bcrypt 验证密码
    -> 如果开启 TOTP，校验验证码
    -> 生成 access token / refresh token
    -> 设置 HttpOnly Cookie
```

规则：

- 用户不存在、密码错误统一返回认证失败
- 未激活用户禁止登录
- 开启 TOTP 后必须传 `totp_code`
- 登录成功更新 `last_login_at`

## 钱包登录

流程：

```text
POST /api/auth/wallet/nonce
    -> 生成 nonce 和待签名 message

POST /api/auth/wallet/login
    -> 验证 EIP-191 签名
    -> 根据钱包地址查找用户
    -> 不存在则自动创建钱包用户
    -> 生成 JWT Cookie
```

当前注意点：

- `AuthService.get_wallet_nonce` 中有 TODO：nonce 应存入 Redis 并设置 5 分钟过期。
- 当前钱包登录验证依赖提交的 `message`，后续应改为服务端校验 nonce 防重放。

## TOTP

TOTP secret 生成后使用 Fernet 加密存储。

流程：

```text
setup -> 生成 secret + otpauth uri -> 加密 secret 写入 users.totp_secret
verify -> 解密 secret -> 校验 6 位验证码 -> 设置 totp_enabled = true
disable -> 关闭 TOTP
```

配置：

- `SECRET_KEY`
- `TOTP_ENCRYPTION_SALT`
- `TOTP_ISSUER`

## API Key

创建流程：

```text
POST /api/auth/api-keys
    -> 生成 key
    -> 生成 secret
    -> bcrypt hash secret
    -> 保存 key + secret_hash
    -> 明文 secret 仅返回一次
```

认证流程：

```text
X-Api-Key + X-Api-Secret
    -> 根据 key 查询 active api_keys
    -> 检查 expires_at
    -> bcrypt 校验 secret
    -> 更新 last_used_at
    -> 返回 active user
```

## 依赖注入

认证依赖位于 `app/api/auth/dependencies.py`。

| 依赖 | 说明 |
| --- | --- |
| `get_current_user` | JWT Cookie/Header |
| `get_current_active_user` | JWT + active |
| `get_current_superuser` | JWT + superuser |
| `get_current_user_from_api_key` | API Key + Secret |
| `get_current_user_flexible` | 优先 JWT，其次 API Key |
