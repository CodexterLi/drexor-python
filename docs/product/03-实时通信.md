# 实时通信

## 当前能力

系统提供两个 WebSocket 入口：

| 路径 | 能力 |
| --- | --- |
| `/ws` | 建立连接、心跳、消息 echo |
| `/ws/broadcast` | 建立连接、广播消息 |

## 消息类型

| 类型 | 说明 |
| --- | --- |
| `welcome` | 连接成功 |
| `ping` | 心跳请求 |
| `pong` | 心跳响应 |
| `message` | 普通消息 |
| `echo` | 单连接回声 |
| `broadcast` | 广播消息 |
| `error` | 错误消息 |

## 当前限制

- 连接状态保存在单进程内存中。
- 多实例部署时，广播不会跨进程同步。
- 目前没有绑定用户身份和个人频道。

后续如果需要跨实例广播，应接入 Redis Pub/Sub 或 Redis Streams。
